## Project Guardrails

* **Версии**: Next.js **14.x** + `next-intl@3`. Не обновлять мажорно без отдельного этапа. Tailwind — без кастомной сборки. 
* **Архитектура**: только App Router. Не переносить в `pages/`, не ломать существующие роуты.
* **i18n**: любые новые строки **обязаны** быть добавлены в `messages/en.json` и `messages/uk.json`. Сборка без `MISSING_MESSAGE`. 
* **UI-инварианты**: градиентный фон в лэйаутах; переключатель языка видим на всех брейкпоинтах; hero-текст и кнопки — по центру <768px, слева ≥768px; логотипы в hero — одинаковая высота.
* **Безопасность**: секреты только в ENV; `.env.local.example` обновляем, секреты в гите — запрещено.
* **Ничего не удалять** из текущего функционала без явного согласования (UI/страницы/роуты/контент).

## PR Policy

* Один **малый, тематический PR** на шаг (этап/подэтап) — не смешивать CMS/платежи/SEO. 
* В PR указывать: цель, список файлов по путям, как проверить (команды), какие ENV нужны.
* Запрещено коммитить `.env.local`, ключи, приватные ID.
* Коммиты — **Conventional Commits**: `feat(...)`, `fix(...)`, `chore(...)`.

## Definition of Done (общий)

* `npm run build && npm start` — без ошибок и предупреждений i18n.
* UI на `/uk` и `/en` визуально соответствует макетам и инвариантам.
* Локализация, доступность (alt, aria) — соблюдены.
* Нет регрессий Lighthouse/A11y (цели — см. общий план KPI). 

## Testing & CI

* Локально: `npm i`, `npm run build`, `npm start`.
* На этапе CMS/Payments дальнейшие шаги добавят smoke-тесты (Playwright) и GA события (см. план). 

## ENV & Secrets

* В каждом PR, где появляются новые переменные, обновлять `.env.local.example` и README (раздел Setup).
* Любые платёжные/почтовые токены — только в окружении (Vercel/Render), не в коде.
